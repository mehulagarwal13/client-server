Build a complete, fully functional mentorship platform with three user roles: Student, Mentor, and Admin. 
Design every part of the system in a clean, modern, intuitive, and user-friendly format. 
The platform should include registration flows, authentication, role-based dashboards, community groups, 
real-time chat spaces, and management sections for announcements, users, sessions, and profiles.

-------------------------------------------------------------------
PART 1 — AUTHENTICATION & ROLE SELECTION
-------------------------------------------------------------------

Create the following:

1. A login page where users can enter email and password.
2. A role selection screen allowing new users to choose:
   - “Register as Student”
   - “Register as Mentor”
3. After successful registration, users should be redirected to the dashboard meant for their role.
4. Admin accounts are pre-created and managed separately.
5. Include friendly error messages, password visibility toggle, and validation for all required fields.

-------------------------------------------------------------------
PART 2 — MENTOR REGISTRATION (EXTREMELY DETAILED FORM)
-------------------------------------------------------------------

Create a highly polished, multi-step Mentor Registration experience. 
Include ALL of these fields in a clean and organized layout:

Step 1 — Basic Profile
- Mentor Full Name  
- Official Email ID  
- Mobile Number  
- Profile Photo Upload  
- LinkedIn Profile URL  
- Portfolio / GitHub URL  
- Password  

Step 2 — Professional Background
- Current Job Title  
- Current Company  
- Years of Experience  
- Skills / Expertise (multiple selectable items)  
- Languages the mentor can speak  

Step 3 — Mentorship Information
- Mentorship Areas (multiple selectable categories)  
- Availability Hours (days of the week + preferred time slots)  
- Short Bio describing the mentor  
- Additional notes or optional introduction  

Each step should have:
- Clear navigation buttons (Next, Back, Submit)
- Real-time field validation
- A progress tracker at the top showing how many steps are completed

PART 4 — STUDENT DASHBOARD (FULL DESCRIPTION)
-------------------------------------------------------------------

After login, create a completely personalized Student Dashboard with:

1. A welcome banner showing the student’s name.
2. A quick snapshot section showing:
   - Number of enrolled mentorship programs
   - Lessons or sessions completed
   - Active learning streak
3. A progress tracker for different learning roadmaps like:
   - Web Development
   - DSA
   - Career Skills  
   (Each with progress bars and status labels such as Beginner, Intermediate, or Advanced.)
4. A section displaying upcoming mentorship sessions with:
   - Session title  
   - Mentor name  
   - Date and time  
   - Join button  
5. A dedicated announcements section showing:
   - Important updates  
   - Mentor posts  
   - New resources uploaded  
6. A mini preview of the community chat showing the last few messages from the main channel.
7. Quick action buttons:
   - Study Material  
   - Assignments  
   - Community Chat  
   - Profile  
   - Settings  

Everything should be neatly organized, clean, and highly readable.

PART 5 — MENTOR DASHBOARD (FULL DESCRIPTION)
-------------------------------------------------------------------

The Mentor Dashboard should allow mentors to manage mentees, sessions, doubts, and communication.

Sections to include:

1. A personalized welcome header with mentor name.
2. Overview cards showing:
   - Total mentees assigned  
   - Upcoming mentorship sessions  
   - Pending doubts/questions from students  
   - Messages received today  
3. A full mentee list with:
   - Student name  
   - Email  
   - Interest area  
   - Last active time  
   - Buttons: View Profile, Message  
4. Session Management:
   - Table to view all upcoming sessions  
   - A form to create new sessions with fields:
     * Title  
     * Date and Time  
     * Mode (online/offline)  
     * Meeting link  
   - Buttons to edit or cancel existing sessions  
5. Doubt Management:
   - List of questions asked by students  
   - Mentor can click “Reply” to answer through chat or comments  
6. Community Quick Access:
   - List of groups the mentor is part of  
   - Each with a shortcut button: “Open Chat”
